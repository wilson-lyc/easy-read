## 2. 模块设计

### 2.1 视图层

| 组件 | 职责 | 技术方案 |
|------|------|---------|
| **TranslationTooltip** | 划词翻译后的浮动气泡，显示译文 + 操作按钮（支持 Markdown 编辑器和 PDF 视图） | Obsidian `Plugin.registerDomEvent` + 坐标定位计算 |
| **TranslationModal** | 全文翻译的设置弹窗（语言选择、输出方式等） | Obsidian `Modal` 子类 |
| **TranslationView** | 双语对照结果展示面板（可选侧边栏视图） | Obsidian `ItemView` 子类（注册到左侧栏） |
| **PDFTooltipOverlay** | PDF 视图中专用的浮动翻译结果层，解决 PDF 页面坐标与编辑器坐标差异，支持拖拽避免遮挡原文 | 基于 PDF 视口坐标定位，独立 DOM 容器 |

### 2.2 核心服务层

#### 翻译引擎（Translator）
```
class Translator {
  async translate(text: string, options: TranslateOptions): Promise<TranslateResult>
  async translateStream(text: string, options: TranslateOptions): AsyncIterable<string>
  cancel(): void
}
```
- 负责组合 Chunker + Provider 完成翻译
- 支持普通和流式两种模式
- 调用 Stream 时返回 AsyncIterable

#### 文本分块器（Chunker）
- **职责**：将长文本按语义/段落拆分为适合 LLM Context 的块
- **策略**：
  - 按 Markdown 标题分块（`#`, `##`, `---`）
  - 按段落（双换行）分块
  - 按句子分块（作为后备）
  - 每块预留 10% 重叠（Overlap）以保语义连贯
- **接口**：
  ```
  class Chunker {
    chunk(text: string, maxTokens: number): Chunk[]
    reconstruct(chunks: TranslatedChunk[]): string
  }
  ```

#### 缓存管理（CacheManager）
- **存储**：使用 Obsidian 的 `Plugin.loadData()/saveData()` 或 IndexedDB
- **缓存 Key**：`hash(sourceText + sourceLang + targetLang + modelId)`
- **有效期**：默认 24 小时，可在设置中配置
- **最大条目数**：5000 条，超出后 LRU 淘汰

#### PDF 解析器（PDFParser）
- **职责**：处理 Obsidian PDF 预览视图中选中文本的提取与清洗
- **核心能力**：
  - 监听 PDF 视图的选中事件（`selectionchange`），获取选中文本
  - **连字符处理**：合并跨行断词（如 `transla-\ntion` → `translation`）
  - **换行规范化**：将人工换行替换为空格，段落双换行保留
  - **Unicode 清理**：处理特殊空格、软连字（soft hyphen）、零宽字符等
  - **文本位置映射**：记录选中文本在 PDF 页面上的坐标范围，用于 Tooltip 定位
- **接口**：
  ```
  class PDFParser {
    isPDFView(leaf: WorkspaceLeaf): boolean
    getSelectionText(): string
    getSelectionPosition(): { x: number, y: number, page: number }
    cleanText(raw: string): string
  }
  ```
- **注意**：PDF 文本提取依赖 Obsidian 内置 PDF 渲染引擎的 Selection API，不引入第三方 PDF 解析库。对于扫描版 PDF（图片型），暂时不支持（留作 P3 的 OCR 需求）。

### 2.3 LLM Provider 抽象层

```
interface LLMProvider {
  readonly id: string
  readonly name: string
  readonly supportsStreaming: boolean

  translate(request: TranslateRequest): Promise<TranslateResponse>
  translateStream(request: TranslateRequest): AsyncIterable<string>
  estimateTokens(text: string): number
}
```

**各 Adapter 实现要点：**

| Provider | API 基础 | 认证方式 | 注意事项 |
|----------|---------|---------|---------|
| OpenAI | `chat/completions` | API Key | GPT-4o-mini 性价比最优 |
| Anthropic | `messages` | API Key | Claude 3.5 Sonnet 翻译质量佳 |
| Azure OpenAI | `openai/deployments/{name}` | API Key + Endpoint | 需额外配置 Deployment Name |
| Ollama | 本地 HTTP API | 无需认证 | 需配置 Base URL + Model Name |
| 自定义 | 用户配置 | 取决于实现 | 预留扩展点 |

### 2.4 设置面板

**配置项分类：**

```
├── LLM 提供商
│   ├── 启用提供商（下拉选择主用/备用）     → string[]
│   ├── Provider 配置（每个 Provider 动态表单）
│   │   ├── API Key / Authentication        → string (加密存储)
│   │   ├── Base URL / Endpoint             → string
│   │   ├── Model Name                      → string
│   │   └── 高级参数（Temperature, Max Tokens） → number
│   └── 流式输出开关                        → boolean
│
├── 翻译偏好
│   ├── 源语言（自动检测/指定）             → string
│   ├── 目标语言                           → string
│   └── 自定义 System Prompt               → string (textarea)
│
├── 划词翻译
│   ├── 触发方式（快捷键/浮动按钮/右键菜单） → string[]
│   ├── 快捷键绑定                         → Hotkey
│   ├── 自动显示翻译气泡                   → boolean
│   └── PDF 支持                           → boolean (启用/禁用 PDF 划词)
│
├── PDF 设置
│   ├── 连字符自动合并                     → boolean
│   ├── 行首行尾空格修剪                   → boolean
│   └── Tooltip 默认位置（跟随选中/固定在角落） → string
│
├── 全文翻译
│   ├── 默认输出方式（覆盖/新建/双语对照）   → string
│   └── 分块大小（Token）                   → number
│
├── 缓存与隐私
│   ├── 启用缓存                           → boolean
│   ├── 缓存有效期                         → number (minutes)
│   └── 本地模式（仅使用 Ollama）            → boolean
│
└── 开发者选项
    ├── 调试日志                            → boolean
    └── 导出日志                            → button
```

**API Key 安全存储**：使用 `Plugin.loadData()` 存储到 Obsidian 本地 `data.json`，文件处于 Obsidian 配置目录下（通常权限 600）。如需额外保护，可提示用户使用系统密钥链（macOS Keychain），留作 P2。

### 2.5 System Prompt 设计

**默认翻译 Prompt：**

```
你是专业翻译。请将以下文本从 {sourceLang} 翻译为 {targetLang}。

要求：
1. 保留原文的所有 Markdown 格式（如 **粗体**、`行内代码`、[链接](url)）
2. 专业术语请使用行业内通用译法
3. 诗歌/文学作品应保留原文韵律感并注明
4. 仅输出翻译结果，不要添加解释

原文：
{text}
```

> PDF 场景使用相同 Prompt。PDF 文本在传入 LLM 前已由 PDFParser 完成清洗，无需额外处理。

用户可完全自定义此 Prompt，在设置中覆盖。

## 3. 数据流

### 3.1 划词翻译数据流（Markdown）

```
用户选中文本
    │
    ▼
Obsidian Editor 获取 selection
    │
    ▼
检查缓存 (CacheManager.get)
    │
    ├── Hit ──→ 直接显示
    │
    └── Miss
         │
         ▼
    Token 估算 (estimateTokens)
         │
         ▼
    发送到 LLM Provider (调用 translate)
         │
         ▼
    写入缓存 (CacheManager.set)
         │
         ▼
    渲染 Tooltip 组件（支持替换原文）
```

### 3.2 划词翻译数据流（PDF）

```
用户在 PDF 视图中选中文本
    │
    ▼
PDFParser 监听 leaf.view.getViewType() === 'pdf'
    │
    ▼
PDFParser.getSelectionText() —— 获取原始文本
    │
    ▼
PDFParser.cleanText() —— 清洗（去连字符、规整换行、清理 Unicode）
    │
    ▼
检查缓存 (CacheManager.get)
    │
    ├── Hit ──→ 直接显示
    │
    └── Miss
         │
         ▼
    发送到 LLM Provider (调用 translate)
         │
         ▼
    写入缓存 (CacheManager.set)
         │
         ▼
    PDFTooltipOverlay 根据选中坐标定位并渲染
    （独立于编辑器 DOM 的浮动容器）
```

### 3.3 全文翻译数据流（Markdown）

```
用户触发全文翻译
    │
    ▼
解析当前笔记内容 (this.app.vault.read)
    │
    ▼
Chunker 分块
    │
    ▼
逐块调用 LLM Provider（队列 + 速率限制）
    │
    ▼
Chunker.reconstruct 拼接结果
    │
    ▼
根据输出模式处理：
    ├── 覆盖原文 → this.app.vault.modify
    ├── 新建笔记 → this.app.vault.create (文件名加 [EN/CN] 后缀)
    └── 双语对照 → 渲染 TranslationView
```

## 4. 错误处理与容错

| 错误类型 | 处理策略 |
|---------|---------|
| API 超时 | 重试 2 次（指数退避：1s → 3s） |
| API 返回格式异常 | 记录原始响应到日志，通知用户 |
| 网络断开 | 提示用户检查网络，保留本地缓存可读 |
| Token 超限 | 自动降低分块大小重试 |
| Provider 不可用 | 自动切换到备用 Provider（如有配置） |
| 无效 API Key | 清除该 Provider 的令牌缓存，提示重新配置 |
| **PDF 选中为空** | 检测选中文本是否为空或纯空白字符，不发起翻译请求 |
| **PDF 文本提取异常** | 回退到浏览器的 window.getSelection() 原始文本，记录日志 |

## 5. 性能设计

- **懒加载**：插件在 Obsidian 启动时不主动初始化 LLM Client，仅在用户首次触发翻译时才加载
- **Web Worker（P2）**：Token 估算等 CPU 密集型操作可移至 Web Worker 避免阻塞 UI
- **分块并发**：全文翻译的多个块可并行发送（在 Rate Limit 约束下），通过 `Promise.all` 控制并发数（默认 3）
- **节流**：划词翻译快速连续选中时，用 300ms Debounce 避免无效请求
- **PDF 事件隔离**：PDF 视图中只监听 `selectionchange` 事件，不注入编辑器相关逻辑，避免与 Markdown 视图冲突

## 6. 技术选型

| 技术/工具 | 选型 | 原因 |
|----------|------|------|
| 语言 | TypeScript | Obsidian 插件首选，类型安全 |
| 构建工具 | esbuild + obsidian-plugin-template | Obsidian 社区推荐 |
| UI 组件 | Obsidian API (原生) | 避免额外依赖，保持主题一致 |
| API 客户端 | fetch (原生) | 各 LLM Provider 均支持 HTTP/Stream，无需额外库 |
| PDF 文本提取 | Obsidian 内置 PDF Viewer Selection API | 不引入第三方 PDF 库，轻量化 |
| Token 估算 | tiktoken (社区版) / 自研启发式 | 用于分块策略 |
| 测试 | Vitest + Obsidian 测试工具链 | 单元测试核心逻辑 |
| 发布 | GitHub Releases + BRAT | Obsidian 社区插件发布流程 |
