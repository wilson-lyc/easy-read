# EasyRead — 开发路线图（Agent 版）

## 总览

| 阶段 | 目标 | 里程碑产出 | 前置依赖 |
|------|------|-----------|---------|
| **P0: MVP** | 划词翻译（Markdown + PDF）+ 全文翻译 + OpenAI Provider | Alpha 内测版 (BRAT) | — |
| **P1: 体验增强** | 多 Provider、双语对照、缓存、Markdown 保留 | Beta 公测版 | P0 ✅ |
| **P2: 生态完善** | 批量翻译、术语表、性能优化、i18n | v1.0 正式发布 | P1 ✅ |
| **P3: 持续迭代** | 社区反馈驱动、高级特性 | 社区插件上架 | P2 ✅ |

> 所有阶段按依赖顺序执行。AI Agent 24h 不间断执行，任务以 S（开始）→ E（结束，具体验证条件）标记完成。

---

## Phase 0 — MVP

**目标**：在 Obsidian 内实现划词翻译（Markdown + PDF）和全文翻译闭环，支持 OpenAI 作为首个 LLM Provider。

**入口条件**：无（初始项目）。

---

### Task-0.1: 项目骨架搭建

```
S → 创建项目目录结构
    ├── package.json           # TypeScript + esbuild 构建配置
    ├── tsconfig.json          # 严格模式配置
    ├── manifest.json          # Obsidian 插件清单（id: easy-read, name: EasyRead）
    ├── esbuild.config.mjs     # 开发/生产构建脚本
    ├── .gitignore
    └── src/
        ├── main.ts            # 插件入口（空白注册）
        └── manifest.json
E → `npm run build` 产出 main.js + manifest.json，Obsidian 社区插件规范校验通过
```

### Task-0.2: 设置系统

```
S → 实现 src/settings/*
    ├── defaults.ts            # 默认配置常量
    ├── SettingsManager.ts     # 封装 loadData/saveData，提供类型安全的读写
    └── SettingsTab.ts         # Obsidian PluginSettingTab 子类
                                ├── API Key 输入框（password 类型）
                                ├── Model 选择/输入
                                └── 目标语言选择
E → 插件在 Obsidian 设置页出现 "EasyRead" 面板，可填写 API Key 并保存
```

### Task-0.3: LLM Provider 接口 + OpenAI 适配器

```
S → 实现 src/providers/
    ├── interface.ts           # LLMProvider 接口定义
    │     class LLMProvider {
    │       readonly id: string
    │       readonly name: string
    │       readonly supportsStreaming: boolean
    │       translate(request: TranslateRequest): Promise<TranslateResponse>
    │       translateStream(request: TranslateRequest): AsyncIterable<string>
    │       estimateTokens(text: string): number
    │     }
    ├── OpenAIAdapter.ts       # 适配 OpenAI chat/completions API
    │                          # 支持普通 + 流式 (SSE)
    │                          # 默认模型: gpt-4o-mini
    └── index.ts               # Provider 注册表
E → OpenAIAdapter.translate("Hello") 返回翻译结果 { translatedText: "你好" }
E → translateStream 可逐块输出流式结果
E → 单元测试覆盖正常响应、空响应、错误响应
```

### Task-0.4: 文本分块器

```
S → 实现 src/core/translator/Chunker.ts
    ├── chunk(text, maxTokens): Chunk[]
    │    策略优先级：
    │      1. 按 Markdown 标题分块（# / ## / ---）
    │      2. 按段落（双换行 \n\n）分块
    │      3. 按句子（。！？.!?）分块
    │    每块尾部预留 10% token 的 Overlap 保语义连贯
    ├── reconstruct(translatedChunks): string
    └── types.ts
E → "A\n\nB\n\nC" (maxTokens=1) → 3 个 Chunk，每个尾部含 10% Overlap
E → reconstruct 后与原文段落数一致
E → 单元测试：纯文本、Markdown（标题/表格/代码块）、超长文本
```

### Task-0.5: 划词翻译（Markdown）

```
S → 实现 src/ui/TranslationTooltip.ts
    ├── 监听 Markdown Editor 的 mouseup/selectionchange
    ├── 选中 > 0 字符后显示浮动按钮
    ├── 点击按钮/快捷键 Ctrl+T 触发翻译（调用 Translator）
    ├── 结果显示在 Tooltip 气泡中（定位在选区附近）
    ├── 气泡操作：Esc 关闭、点击"替换原文"将译文写回编辑器
    └── 300ms Debounce 防止快速连点

S → 实现 src/core/translator/Translator.ts
    ├── translate(text, options) → 调用 LLMProvider
    └── cancel() → 中断进行中的请求

S → 快捷键 Ctrl+T 注册到 Obsidian Hotkey 系统
S → 右键菜单注册 "EasyRead: 翻译选中文本"
E → 在 Markdown 编辑器中选中英文文本 → 触发翻译 → 气泡显示中文译文
E → 点击"替换原文" → 选中文本被译文覆盖
E → 连续快速选中只发起一次请求
```

### Task-0.6: 全文翻译

```
S → 实现 src/ui/TranslationModal.ts
    ├── 语言选择：源语言（auto/指定）、目标语言
    ├── 输出方式：覆盖原文 / 新建笔记 / 双语对照（P1 实现）
    └── 确认按钮触发翻译

S → 实现请求队列 src/utils/requestQueue.ts
    ├── 并发控制（默认 3）
    ├── 指数退避重试（1s → 3s → 放弃）
    └── 取消信号

S → 实现 src/utils/tokenEstimator.ts（启发式：中 2 char/token，英 1 token/4 char）

S → 全文翻译主流程：
    ├── 读取笔记内容 (app.vault.read)
    ├── Chunker.chunk() 分块
    ├── 逐块调用 Provider（通过请求队列）
    ├── Chunker.reconstruct() 拼接
    └── 根据输出模式写入 (modify / create)

S → 注册命令 "EasyRead: 全文翻译"
E → 5000 词英文笔记全文翻译为中文，新建笔记内容完整
E → 覆盖原文模式正确写入
E → API 中断后重试 2 次，成功后继续
```

### Task-0.7: PDF 划词翻译

```
S → 实现 src/core/pdf/PDFParser.ts
    ├── isPDFView(leaf): boolean    — 检测当前 leaf 是否为 PDF 视图
    ├── getSelectionText(): string  — 获取选中文本（依赖 Obsidian PDF 内置 Selection API）
    ├── cleanText(raw): string      — 清理管线：
    │                                  1. 合并连字符断词（trans-\nlation → translation）
    │                                  2. 换行替换为空格（保留段落双换行）
    │                                  3. 移除软连字 (U+00AD) 和零宽字符
    │                                  4. 首尾空白修剪
    └── getSelectionPosition(): PDFPosition
        └── { x, y, page }         — PDF 视口坐标，用于 Tooltip 定位

S → 实现 src/ui/PDFTooltipOverlay.ts
    ├── 监听 leaf.view.getViewType() === 'pdf' 下的 selectionchange
    ├── 选中后出现浮动按钮（叠加在 PDF 页面上方）
    ├── 触发翻译 → Tooltip 基于 PDFPosition 定位渲染
    ├── 支持拖拽（避免遮挡原文）
    └── Esc 关闭

S → PDF 划词右键菜单注册
E → 在 Obsidian 中打开 PDF 文件 → 选中英文段落 → 浮动按钮出现
E → 点击翻译 → 气泡显示译文，可拖拽
E → 跨页选中文本正常提取
E → 扫描版 PDF（图片型）提示"当前 PDF 不支持文本选择"
```

### Task-0.8: Alpha 发布

```
S → 完善 main.ts：注册所有命令、视图、设置
S → 错误处理全局化：API 超时/Key 无效/网络断开 → 用户通知
S → GitHub Release 打包 + BRAT 兼容
E → Alpha 内测版本可安装
E → 交付物清单：
    ├── BRAT 可安装的 Alpha 版
    ├── 划词翻译（Markdown + PDF，OpenAI 英文→中文）
    ├── 全文翻译（≤5000 词）
    └── 基础设置面板（API Key + Model + 目标语言 + PDF 开关）
```

## Phase 1 — 体验增强

**目标**：多 LLM 提供商、双语对照、翻译缓存、Markdown 保留。

**入口条件**：Phase 0 全部 Task E 条件满足 ✅

---

### Task-1.1: 多 Provider 适配

```
S → 实现 AnthropicAdapter
    ├── Messages API + Stream
    └── 默认模型: claude-sonnet-4-20250514

S → 实现 OllamaAdapter
    ├── 本地 HTTP API (无需认证)
    └── 可配置 Base URL + Model Name

S → 设置面板动态 Provider 配置表单
    ├── Provider 选择器（主用 + 备用）
    └── 每个 Provider 独立的 API Key/URL/Model 字段

S → Provider 自动切换
    ├── 主用失败 → 自动切换到备用
    └── 切换通知用户

E → OpenAI / Anthropic / Ollama 均可发起翻译请求
E → API Key 无效时自动切换到备用 Provider
```

### Task-1.2: 翻译缓存

```
S → 实现 src/core/cache/CacheManager.ts
    ├── 存储：Plugin.loadData/saveData 或 IndexedDB
    ├── Key: hash(sourceText + sourceLang + targetLang + modelId)
    ├── TTL: 默认 24h（设置中可配）
    ├── 最大条目：5000，超出 LRU 淘汰
    ├── 缓存穿透保护：同一 Key 多个并发请求只发一次 API
    └── 缓存统计：命中/未命中次数展示在设置页
E → 同一文本 5 秒内再次翻译 → 命中缓存，不发起 API
E → 手动清除缓存按钮可用
```

### Task-1.3: 双语对照视图

```
S → 实现 src/ui/TranslationView.ts (ItemView 子类)
    ├── 注册到 Obsidian 左侧栏（图标：book-open 或类似）
    ├── 支持两种布局：
    │   ├── 交错：原文段落 ↔ 译文段落交替
    │   └── 分栏：左右两栏并排
    ├── 与全文翻译联动：输出方式选择"双语对照"时自动打开此视图
    └── 支持 Markdown 渲染
E → 执行全文翻译 → 选择"双语对照" → 侧边栏打开，原文/译文对照显示
```

### Task-1.4: Markdown 格式保留

```
S → 优化 System Prompt 明确要求保留 Markdown
S → 后处理管线 src/core/postprocess/
    ├── 修复 LLM 破坏的 Markdown 格式
    │   ├── 链接 [text](url) → 确保括号完整
    │   ├── 代码块 ``` → 确保三重反引号成对
    │   ├── 表格 | --- | --- | → 确保分隔行存在
    │   └── 高亮 ==text== → 保留
    └── 与 Chunker.reconstruct 集成

S → 实现 PromptManager
    ├── 读取用户自定义 System Prompt
    ├── 合并模板 + 用户 Prompt
    └── 设置面板中提供 TextArea 编辑器
E → 含链接/表格/代码块的笔记翻译后 Markdown 结构完整
E → 用户可自定义 Prompt 并生效
```

### Task-1.5: 体验打磨

```
S → Token 估算器升级（tiktoken 启发式或 wasm 版）
S → 全文翻译流式写入（逐块返回时逐步写入编辑器，非一次性）
S → UI 主题适配（亮色/暗色模式分别测试）
S → 重试机制完善（所有 Provider 统一的指数退避策略）
S → PDF 边界情况处理：
    ├── 跨页选中 → 合并多页选择文本
    ├── 旋转 PDF → 坐标转换
    └── 扫描件 → 检测并回退提示
S → PDFTooltipOverlay 拖拽位置持久化（localStorage）
S → 完善错误通知（Obsidian Notice + 状态栏提示）
E → Beta 公测版本可安装
E → 交付物清单：
    ├── Beta 版（OpenAI + Anthropic + Ollama）
    ├── 划词翻译（Markdown + PDF，含连字处理）
    ├── 全文翻译 + 双语对照视图
    ├── 翻译缓存
    ├── Markdown 格式保留
    ├── 用户自定义 Prompt
    └── 用户文档（README + 使用说明）
```

```

    ├── 所有核心路径 E2E

## Phase 3 — 持续迭代

**入口条件**：Phase 2 全部 Task E 条件满足 ✅，社区反馈开始输入。

---

### 候选特性池（按优先级排序）

| 优先级 | 特性 | 说明 |
|--------|------|------|
| P0 | 自定义 Provider | 用户配置自己的 API Endpoint（兼容 OpenAI API 格式） |
| P1 | 翻译历史 | 查看/搜索历史翻译记录，可复用 |
| P1 | 词典模式 | 选中单词显示释义 + 例句（不翻译整句） |
| P2 | OCR 翻译 | 图片/扫描版 PDF 文字识别 + 翻译 |
| P2 | PDF 全文翻译 | PDF 整篇提取 → 翻译 → 保存为新 Markdown 笔记 |
| P2 | 快捷键自定义 | 注册到 Obsidian 原生 Hotkey 系统 |
| P3 | Templater/Dataview 集成 | 提供翻译命令/函数供其他插件调用 |

### 长期维护规则

```
- 每个社区 Issue 进入候选池，按 👍 数排序
- P0-P1 issue 持续处理，按优先级从候选池提取
- LLM 新模型发布 → 更新默认配置并发布补丁
- 社区 PR → 24h 内 Review
```
